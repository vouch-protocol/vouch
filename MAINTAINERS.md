## Roles

### Lead Maintainer
- Full write access to all repositories
- Release authority for all packages
- Final decision on architecture and roadmap
- Code of Conduct enforcement

### Maintainer
- Write access to specific modules
- Can approve and merge PRs in their area
- Participates in release process
- Mentors contributors

### Committer
- Write access to specific areas
- Can approve PRs (requires maintainer merge)
- Active code contributor

## Becoming a Maintainer

Contributors who have made significant contributions to the project may be invited to become maintainers. The process:

1. **Demonstrate expertise** - Multiple accepted PRs in a specific area
2. **Show commitment** - Consistent participation over 3+ months
3. **Nomination** - Nominated by existing maintainer
4. **Approval** - Approved by Lead Maintainer
