Certificate Practice Statement

Vouch Protocol C2PA Certificate Authority

Version 2.0 March 2026 OID: 1.3.6.1.4.1.62558.1.1

1. Overview

This Certificate Practice Statement (CPS) describes the practices and procedures of the Vouch Protocol C2PA Certificate Authority (CA). The Vouch Protocol CA issues digital certificates for Content Credentials signing in accordance with the C2PA Certificate Policy (OID: 1.3.6.1.4.1.62558.1.1).

The CA is operated by Vouch Protocol and provides certificates to subscribers including Cygn (cygn.me) and other entities requiring C2PA content credential signing capabilities. Vouch Protocol is a member of C2PA, W3C, and the Linux Foundation.

2. CA Organization

  • CA Operator: Vouch Protocol (vouch-protocol.com)
  • Governing Body: Vouch Protocol Maintainers
  • CA Contact: ca@vouch-protocol.com
  • CPS Location: https://vouch-protocol.com/cps.html
  • Subscribers: Cygn Weaves (cygn.me) and approved entities

3. Certificate Profiles

3.1 Trust Chain Architecture

The Vouch Protocol CA operates a three-tier certificate hierarchy with ECDSA P-384 for CA certificates and Ed25519 for leaf signing certificates:

  • Vouch Protocol Root CA (ECDSA P-384, self-signed, 20-year validity)
  • Vouch Protocol Intermediate CA (ECDSA P-384, signed by Root CA, 5-year validity)
  • Leaf Signing Certificates (Ed25519, signed by Intermediate CA, 366-day or 90-day validity)

4. Key Management

4.1 Key Generation

All CA keys (Root and Intermediate) are generated in Google Cloud KMS HSM, which is FIPS 140-2 Level 3 certified. Keys are generated as non-exportable asymmetric signing keys using ECDSA P-384 (EC_SIGN_P384_SHA384).

Leaf signing keys use Ed25519 and may be generated locally or within HSM depending on the subscriber deployment model.

4.2 Key Storage

  • Root CA key: Google Cloud KMS HSM, non-exportable, HSM protection level
  • Intermediate CA key: Google Cloud KMS HSM, non-exportable, HSM protection level
  • Leaf signing key: Subscriber-managed (environment variables, local keystore, or HSM)
  • Certificate chain: Published at /.well-known/c2pa-trust-list

4.3 Key Ceremony

Key generation follows a formal ceremony procedure with minimum 3 participants (CA Administrator, Witness, and Independent Witness). Full ceremony documentation is available at vouch-protocol.com/key-ceremony.

5. Certificate Lifecycle

  • Root CA: 20-year validity, offline renewal ceremony with independent witness
  • Intermediate CA: Maximum 1,827 days (5 years), signed by Root during key ceremony
  • Leaf AL1: Maximum 366 days, annual renewal
  • Leaf AL2: Maximum 90 days, quarterly renewal
  • Re-authentication: Maximum 398 days for certificate renewal

Revoked certificates are published in the CRL at /api/v1/c2pa/crl and queryable via OCSP at /api/v1/c2pa/ocsp.

6. Revocation

In the event of key compromise or certificate misuse:

  • The compromised certificate serial number is added to the revocation list
  • CRL and OCSP endpoints immediately reflect the revocation
  • A new leaf certificate is generated using the existing intermediate CA
  • Affected subscribers are notified via email
  • The trust list is updated with new certificate information

Revocation requests can be submitted to ca@vouch-protocol.com. Emergency revocation is processed within 24 hours.

7. Audit and Logging

  • All certificate issuance and revocation events are logged with timestamps
  • Google Cloud KMS provides built-in audit logging for all key operations
  • Ceremony artifacts (attestations, PKI lint reports, chain verifications) are retained for 7 years
  • CRL and OCSP query logs are maintained for compliance reporting
  • C2PA signing operations are logged per subscriber with content hash and certificate serial number

8. Compliance

The Vouch Protocol C2PA CA operates in accordance with:

  • C2PA Certificate Policy (OID: 1.3.6.1.4.1.62558.1.1)
  • C2PA Specification v2.x - Content Credentials trust model
  • FIPS 140-2 Level 3 - Hardware security module certification (Google Cloud KMS)
  • FIPS 186-4 - Digital Signature Standard (ECDSA P-384)
  • RFC 5280 - Internet X.509 PKI Certificate and CRL Profile
  • RFC 6960 - Online Certificate Status Protocol (OCSP)
  • RFC 8032 - Edwards-Curve Digital Signature Algorithm (Ed25519)
  • RFC 3161 - Internet X.509 PKI Time-Stamp Protocol (via DigiCert TSA)

Vouch Protocol is a member of C2PA, W3C, and the Linux Foundation. The protocol specifications are being donated to W3C.
