Introducing Vouch Git Workflow: One Command to Sign All Your Commits

Supply chain attacks are on the rise. In 2024 alone, we saw major incidents affecting millions of developers through compromised dependencies and malicious commits. The solution? Cryptographically sign every commit to prove it came from you.

With Vouch Protocol, you can set this up in under 60 seconds.

üìã Prerequisites

  • Python 3.8+ installed
  • Git configured with your name and email
  • An SSH key added to your GitHub account (most developers already have this)

üöÄ Quick Start (60 seconds)

1 Install Vouch Protocol

pip install vouch-protocol

This installs the vouch CLI tool globally.

2 Initialize Your Identity

vouch init

This creates your Vouch identity. You'll be prompted to set a passphrase to encrypt your private key.

Your identity is stored securely in ~/.vouch/keys/.

3 Configure Git Signing

vouch git init

This command does three things automatically:

  • Finds your SSH signing key
  • Configures git config --global to sign commits
  • Sets up the allowed signers file

4 Verify It Works

# Make a test commit
git commit --allow-empty -m "test: verify vouch signing"

# Check the signature
git log --show-signature -1

You should see output indicating the commit is signed.

‚úì Done! All future commits will be automatically signed.

üîç How It Works

Vouch leverages SSH signing (introduced in Git 2.34+). When you run vouch git init, it:

  1. Detects your existing SSH key (typically ~/.ssh/id_ed25519 or ~/.ssh/id_rsa)
  2. Configures Git to use this key for signing: 
    git config --global gpg.format ssh
git config --global user.signingkey ~/.ssh/id_ed25519.pub
git config --global commit.gpgsign true
  3. Creates an allowed signers file for verification

üì± GitHub Integration

For GitHub to show the "Verified" badge:

  1. Go to GitHub Settings ‚Üí SSH Keys
  2. Add your signing key as a Signing Key (not just Authentication)
  3. Push a signed commit‚Äîyou'll see the ‚úÖ badge!
üí° Pro Tip: If you already added your SSH key for authentication, you can use the same key for signing. Just add it again and select "Signing Key" as the type.

üîß Manual Configuration (Alternative)

If you prefer to configure git manually without the CLI:

# Use SSH for signing
git config --global gpg.format ssh

# Set your signing key
git config --global user.signingkey ~/.ssh/id_ed25519.pub

# Auto-sign all commits
git config --global commit.gpgsign true

# Auto-sign all tags
git config --global tag.gpgsign true

üõ†Ô∏è Troubleshooting

Commit fails with "error: Load key failed"

Your SSH key might require a passphrase. Add it to your SSH agent:

eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

GitHub doesn't show "Verified"

Make sure your signing key is added to GitHub as a Signing Key, not just an authentication key.

Check your current git signing config

git config --global --list | grep -E "(gpg|sign)"
