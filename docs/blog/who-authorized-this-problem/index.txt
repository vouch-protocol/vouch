d:T495,<h3>The Pain Point</h3>
<p>Imagine an AI agent buying cloud resources. It uses an API key. If that key is stolen, <em>anyone</em> can buy resources. The logs just show "API Key 123 used." They don't prove <strong>who</strong> used it or <strong>what specific action</strong> they intended. Did the agent mean to buy 1 server or 100? You can't prove it.</p>

<h3>The Solution: "I meant to do exactly this."</h3>
<p>Vouch introduces a "cryptographic sticky note" for every request. Instead of just showing an ID badge (API Key), the agent signs a specific note: <em>"I, Agent Alice, authorize buying 1 server at 2 PM."</em></p>
<p>This note is cryptographically locked. If a hacker intercepts it and tries to change "1 server" to "100 servers," the signature breaks.</p>

<h3>Why It Matters</h3>
<ul><li><strong>Non-Repudiation:</strong> If an agent messes up, it can't say "It wasn't me." The math proves it was.</li>
<li><strong>Granular Control:</strong> Auth isn't just "access to the building," it's "authorized to open door 4."</li>
</ul>
<strong>Key Takeaway:</strong> Move from "who are you?" (Identity) to "who authorized this specific action?" (Intent).
