d:Te44,<h2>What the law actually says</h2>
<p>The EU AI Act's transparency article (Article 50) creates two obligations with hard dates.</p>
<ul><li><strong>From August 2, 2026:</strong> anyone interacting with an AI system must be told they are interacting with an AI. California's AI Transparency Act becomes operative the same day.</li>
<li><strong>From December 2, 2026:</strong> AI-generated or AI-manipulated content must carry machine-readable marking, so software can detect that it is synthetic.</li></ul>
<p>Penalties reach EUR 15M or 3 percent of global turnover. The European Commission's final Code of Practice on marking (June 10, 2026) points to cryptographic content credentials, the C2PA family, as an example of how to do the marking properly.</p>

<h2>Why a label is not enough</h2>
<p>The naive reading of "machine-readable marking" is a metadata tag: a byte in the file that says "AI-generated." The problem is that tags are free to forge and free to strip. Copy the text out of the file and the tag is gone. Add the tag to a human's work and nobody can tell it is a lie.</p>
<p>A signature changes both properties. If the disclosure is a signed credential over the exact bytes of the content, then removing it is detectable (the content arrives without its marking), forging it is infeasible (the signature will not verify), and tampering with the content breaks the binding (the digest no longer matches). The marking stops being a sticker and becomes a property of the content itself.</p>

<h2>The mechanism, in one screen of code</h2>
<p>Vouch Protocol gives an AI agent a cryptographic identity (a DID) and lets it sign structured statements. An AI-origin disclosure is just such a statement: this content, identified by its digest, was produced by an AI, signed by this agent, at this time.</p>
<pre><code>import vouch

agent = vouch.Agent()   # did:key identity, nothing to register
cred  = agent.sign({
    "action":   "disclose",
    "target":   "ai-generated-content",
    "resource": "sha256:&lt;digest of the exact bytes&gt;",
})

ok, who = vouch.verify(cred)   # verifies offline, no server involved</code></pre>
<p>The credential travels with the content. Any reader, platform, or auditor can verify it offline: no account, no API, no phone-home. Change one character of the content and verification fails, because the digest inside the signed statement no longer matches.</p>

<h2>See it run in your browser</h2>
<p>The <a href="/demos/transparency">interactive transparency demo</a> runs this entire flow with real cryptography in your browser: it mints a throwaway agent identity, signs an AI-origin disclosure over a piece of text with the eddsa-jcs-2022 Data Integrity cryptosuite, verifies it live, and shows verification fail the moment you tamper with a single character. Every DID, digest, and signature on that page comes from a live call into the Vouch Protocol core compiled to WebAssembly.</p>

<h2>Where this fits</h2>
<p>The Code of Practice is voluntary and its adequacy assessment is ongoing, so treat all of this as engineering rather than legal advice: what the mechanism above provides is machine-readable, verifiable disclosure of AI origin, which is the technical substance the marking obligation asks for. The same signing primitives extend to images and media through Content Credentials, and to the harder question regulators are already circling: proving which agent took which action, on whose authority.</p>
