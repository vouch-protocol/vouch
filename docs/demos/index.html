Watch an agent get stopped.

You can't read an AI agent's mind, and you can't pre-approve every action it invents. So Vouch does what every institution has always done with authorized actors: it makes each action state its reason, slows down the irreversible ones, and keeps a veto. All three run below, on real credential shapes.

Every verdict here mirrors the shipped SDK modules vouch.reasoning, vouch.deliberation, vouch.caveats, vouch.provenance, and vouch.transparency. Disclosed as PAD-043, PAD-045, PAD-085, and PAD-086.

§ I

Reasoned action

Every action states why, and can't lie about it · vouch.reasoning

Before acting, the agent states why, ties each reason to a real artifact by its hash, and escrows the justification before it executes. An auditor can then prove the reasoning was neither fabricated nor rewritten after the fact.

Try it as the agent
intent delete · /tmp/cache/*
reason user asked to clean /tmp → msg:c-42
escrowed before execution ✓

§ IV

The provenance

Prove the output came from this model on this context · vouch.provenance

The agent binds its output to a fingerprint of the model and a Merkle root over the context it was grounded in. An auditor can re-fetch the sources to reproduce the root, and re-run the model on the same seed to byte-compare the output.

§ V

The transparency log

An action can't be hidden or rewritten · vouch.transparency

Consequential actions land in an append-only Merkle log that signs its size and root. A verifier demands an inclusion proof that an action is in the log, and a monitor demands a consistency proof that a newer tree head extends the older one, so nothing can be omitted or rewritten.

Try it as the monitor
#0 transfer_funds · usd:5000 ✓
#1 publish · blog ✓
#2 delete · db:staging-3 ✓

VERIFIEDinclusion proof holds and the new tree head extends the old
