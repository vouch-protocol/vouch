Cryptographic identity, intent, and continuous trust for autonomous AI agents.

Vouch Protocol is an open, open standard specification for verifying who an agent is, what it intended to do, and whether its runtime state still warrants the authority it was granted. Built on Verifiable Credentials, Decentralized Identifiers, and Data Integrity proofs. Reference implementations in Python, TypeScript, and Go, with an optional hybrid post-quantum profile shipping today.

I - Capabilities

What the protocol provides

Six properties that traditional API keys, OAuth tokens, and bearer credentials cannot provide for autonomous AI agents operating in regulated environments.

I-01

Cryptographic Identity

Every agent holds a Decentralized Identifier (did:web or did:key) backed by an Ed25519 keypair. No API keys, no shared secrets.

§6 Identity Model
I-02

Intent Attestation

Each action is signed as a Verifiable Credential with Data Integrity proofs binding identity, action, target, and resource.

§5 Credential Format
I-03

Resource Delegation

Multi-agent systems gain verifiable principal-to-sub-agent chains. Each link narrows the resource scope.

§9 Delegation Chains
I-04

Identity Sidecar

Private keys never enter the LLM context window. A separate Go binary signs on behalf of the agent over a local IPC channel.

§10 Identity Sidecar
I-05

Heartbeat Renewal

Long-running agents renew SessionVoucher credentials on a periodic schedule. The trust model inverts: untrusted until renewed.

§11 Heartbeat Protocol
I-06

Hybrid Post-Quantum

Cryptosuite hybrid-eddsa-mldsa44-jcs-2026. Ed25519 + ML-DSA-44 sign the same JCS bytes.

§13 Crypto-Agility

For Developers

Getting Vouch into your codebase

Which languages have Vouch SDKs?
[-]

Three reference SDKs: Python (pip install vouch-protocol), TypeScript (npm install @vouch-protocol/core), and Go (go install github.com/vouch-protocol/vouch/go-sidecar/cmd/vouch-sidecar).

All three produce byte-identical credentials thanks to RFC 8785 JCS canonicalization. Cross-language test vectors are published at test-vectors/hybrid-eddsa-mldsa44/. The Python and TypeScript SDKs also expose verification, while the Go binary is primarily a long-running signing daemon (the Identity Sidecar pattern).
